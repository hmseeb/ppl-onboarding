Output: proxy.ts middleware, login page, login API route, and verifyAdminAuth helper function.

  <files>

  <action>
Create `lib/auth.ts` with a `verifyAdminAuth` function:
- Accepts a password string, returns boolean.
- Compares against `process.env.ADMIN_PASSWORD` using `crypto.timingSafeEqual(Buffer.from(input), Buffer.from(expected))`.
- Returns false if ADMIN_PASSWORD env var is not set (fail closed).
- Export a constant `ADMIN_COOKIE_NAME = 'admin_session'`.
- Export a function `generateCookieValue()` that returns a hex string from `crypto.randomBytes(32).toString('hex')`. This value is stored in the cookie and checked against a module-level variable (simple in-memory session). Alternatively, since this is single-password auth, the cookie value can be a known secret derived from the password — use `crypto.createHmac('sha256', process.env.ADMIN_PASSWORD!).update('admin_session').digest('hex')` so it is deterministic and can be verified without server state.
- Export a function `verifySessionCookie(cookieValue: string): boolean` that recomputes the HMAC and compares with `timingSafeEqual`.

Create `app/api/admin/auth/route.ts`:
- POST handler: reads JSON body `{ password: string }`.
- Calls `verifyAdminAuth(password)`. If false, return `Response.json({ error: 'Invalid password' }, { status: 401 })`.
- If true, generate the HMAC cookie value, set `admin_session` cookie with: `httpOnly: true, secure: true, sameSite: 'lax', path: '/', maxAge: 60 * 60 * 24` (24 hours).
- Return `Response.json({ success: true })` with the Set-Cookie header.
- Do NOT use `cookies()` from next/headers (this is a Route Handler, use `new Response` with headers).

CRITICAL: Use `crypto.timingSafeEqual` for ALL comparisons — both password and cookie verification. Never use `===` for secret comparison.
