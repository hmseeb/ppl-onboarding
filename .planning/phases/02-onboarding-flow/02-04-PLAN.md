     before navigating to badaaas.com. A void return type would silently allow

   - Merge broker data with formData to get final display values:

     - Company: formData.company_name ?? broker.company_name
     - Email: formData.email ?? broker.email
     - Phone: formData.phone ?? broker.phone (display with formatPhone)
     - Vertical(s): broker.primary_vertical (and secondary if present) — use Badge
     - Delivery method: formData.delivery_method ?? broker.delivery_method (display as human-readable: "SMS", "Email", "CRM Webhook")
     - Batch size: broker.batch_size
   - Layout — ALL COPY FROM THE SPEC:
     - Headline: "You're All Set, {firstName}. Go Fund Some Deals."
     - Summary Card showing all the merged data above (combining webhook data + onboarding selections)
     - Subtext: "Your {batch_size} referrals are on the way. Watch your {deliveryMethodDisplay} — and remember, speed to lead wins."
       - Where {deliveryMethodDisplay} is a human-readable version of the selected delivery method: "texts" for SMS, "email" for email, "CRM" for crm_webhook
     - Primary CTA: "Go to Dashboard" button (full width, red accent, min-h-[44px])
       - onClick handler MUST await onComplete() before navigating:
         ```
         const handleClick = async () => {
           setIsSubmitting(true)
           await onComplete()
           window.location.href = 'https://badaaas.com'
         }
         ```
         The `await` is critical — without it the browser navigates before the
         POST /api/brokers/[token]/complete call resolves. Because `onComplete`
         returns `Promise<void>`, TypeScript will flag omitting `await` as a
         floating promise (with @typescript-eslint/no-floating-promises or
         similar rules).
