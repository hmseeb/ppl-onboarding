  <name>Task 1: Implement GHL webhook POST endpoint with idempotent upsert</name>
  <files>app/api/webhooks/ghl/route.ts</files>
  <action>
    Create `app/api/webhooks/ghl/route.ts` with a POST handler that:

    1. **Read raw body first** (for future HMAC verification):
       ```typescript
       const raw = await request.text()
       ```
       Then `JSON.parse(raw)` — do NOT use `request.json()` directly. If JSON.parse fails, return 400 with `{ error: 'Invalid JSON' }`.

    2. **Validate with Zod** using `GHLWebhookSchema.safeParse(body)`. If validation fails, return 400 with `{ error: 'Validation failed', details: result.error.flatten() }`. This satisfies HOOK-02.

    3. **Idempotent upsert** using the select-then-insert pattern from 01-RESEARCH.md (Pattern 2):
       - First, SELECT from brokers WHERE ghl_contact_id = payload.ghl_contact_id
       - If exists: return the EXISTING token URL with status "exists" (HOOK-05)
       - If not exists: INSERT new row with `crypto.randomUUID()` as token, normalized data (toTitleCase on first_name/last_name, toLowerCase on email), status 'not_started'
       - If INSERT fails with unique violation (race condition): catch error, SELECT existing row, return its token

       DO NOT use `.upsert()` with `ignoreDuplicates: false` — this overwrites the existing token, breaking the broker's original link (see 01-RESEARCH.md Pitfall 1).

    4. **Normalize data before insert**:
       - `toTitleCase(payload.first_name)` and `toTitleCase(payload.last_name)` from lib/utils/normalize.ts
       - `payload.email.toLowerCase()`
       - Pass through all other fields as-is (phone, company_name, state, verticals are optional/nullable)

    5. **Return response** (HOOK-04):
       ```json
       {
         "onboarding_url": "https://[BASE_URL]/onboard/[token]",
         "broker_name": "[first_name]",
         "status": "created" | "exists"
       }
       ```
       Use `process.env.NEXT_PUBLIC_BASE_URL` for the URL base.

    6. **Error handling**:
       - JSON parse failure: 400
       - Zod validation failure: 400 with details
       - Supabase insert error (non-unique-violation): 500 with `{ error: 'Database error' }` (log the actual error with console.error)
       - NEVER return 5xx for recoverable cases — GHL does not retry on 5xx

    Import from: `@/lib/supabase/server` (createServiceClient), `@/lib/validations/webhook` (GHLWebhookSchema), `@/lib/utils/normalize` (toTitleCase).

    The handler must be THIN — validate, upsert, return. No secondary API calls, no blocking work. Vercel has a 10-second timeout.

    **Part A: Broker Token GET Endpoint (HOOK-06)**

    Create `app/api/brokers/[token]/route.ts` with a GET handler that:

    1. Extract token from dynamic route params. In Next.js 16, params is a Promise:
       ```typescript
       export async function GET(
         _request: NextRequest,
         { params }: { params: Promise<{ token: string }> }
       ) {
         const { token } = await params
       ```

    2. Query Supabase for the broker by token:
       ```typescript
       const supabase = createServiceClient()
       const { data: broker, error } = await supabase
         .from('brokers')
         .select('first_name, last_name, email, phone, company_name, state, primary_vertical, secondary_vertical, batch_size, deal_amount, status, current_step, delivery_method, delivery_email, delivery_phone, crm_webhook_url, contact_hours, weekend_pause, created_at, completed_at')
         .eq('token', token)
         .single()
       ```
       Do NOT select `id` or `ghl_contact_id` — these are internal fields not exposed via the token API.

    3. If broker not found (error or null), return 404: `{ error: 'Broker not found' }`

    4. If found, return the broker data as JSON with 200 status.

    Import from: `@/lib/supabase/server` (createServiceClient), `next/server` (NextRequest).

    **Part B: Health Keepalive Endpoint (INFR-03)**

    Create `app/api/health/route.ts` with a GET handler that:

    1. Creates a Supabase service client
    2. Performs a lightweight query: `supabase.from('brokers').select('id').limit(1)` — this is enough to keep the Supabase project active
    3. Returns 200 with `{ status: 'ok', timestamp: new Date().toISOString() }`
    4. If the query fails, returns 503 with `{ status: 'error', timestamp: new Date().toISOString() }`
    5. Wrap in try/catch for resilience

    This endpoint will be pinged by cron-job.org every 72 hours (3 days) to prevent Supabase free-tier auto-pause at 7 days of inactivity.

    Import from: `@/lib/supabase/server` (createServiceClient).
