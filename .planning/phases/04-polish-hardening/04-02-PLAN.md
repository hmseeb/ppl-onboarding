**In app/layout.tsx:**

Add a `viewport` export (Next.js Metadata API). This generates the viewport meta tag with the `interactive-widget` hint that tells iOS Safari to resize the layout viewport when the keyboard appears.

```tsx
import type { Metadata, Viewport } from "next";

export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  maximumScale: 1,
  interactiveWidget: 'resizes-content',
}
```

The `maximumScale: 1` prevents unintended zoom on input focus (iOS Safari default behavior). The `interactiveWidget: 'resizes-content'` tells the browser to resize the layout viewport when the virtual keyboard appears, so `dvh` units respond to keyboard state.

Do NOT put viewport config inside the `metadata` export -- it must be a separate `viewport` export per Next.js 14+ API.

**In app/globals.css:**

Tailwind CSS v4 includes `min-h-dvh` as a built-in utility (maps to `min-height: 100dvh`). Verify this works by checking the Tailwind v4 docs. If `min-h-dvh` is already available in Tailwind v4, no CSS changes are needed -- just use the class directly.

If for any reason `min-h-dvh` is not recognized by Tailwind v4, add a custom utility in globals.css:
```css
@utility min-h-dvh {
  min-height: 100dvh;
}
```

Tailwind v4 DOES support `min-h-dvh` natively, so the CSS change should NOT be necessary. Only add the utility as a fallback.
