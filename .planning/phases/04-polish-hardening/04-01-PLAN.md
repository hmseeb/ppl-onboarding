  <action>
Rewrite StepTransition.tsx to add:

1. **prefers-reduced-motion support:** Use the `useReducedMotion()` hook from `motion/react`. When reduced motion is preferred, set all animation durations to 0 and disable transforms (opacity-only instant transition, or skip animation entirely).

2. **Smoother transition curve:** Change the current `duration: 0.2` to `duration: 0.25` with `ease: [0.25, 0.1, 0.25, 1]` (standard ease curve). This prevents the jarring snap feeling on mobile.

3. **Prevent layout jumps:** Add `style={{ position: 'relative' }}` to the motion.div wrapper, and use `layout` prop sparingly. The key issue is AnimatePresence mode="wait" already handles this, but ensure the exit animation uses `position: 'absolute'` so the exiting element doesn't push content. Wrap in a div with `style={{ position: 'relative', overflow: 'hidden' }}` to contain the absolute-positioned exiting step.

4. **Keep it simple:** The component should remain a thin wrapper. Do NOT add direction-awareness (tracking forward/back) as it would require prop changes to OnboardingStepper. The current slide-right-in, slide-left-out pattern is sufficient.

Implementation approach:
```tsx
'use client'
import { motion, AnimatePresence, useReducedMotion } from 'motion/react'

interface StepTransitionProps {
  stepKey: number
  children: React.ReactNode
}

export function StepTransition({ stepKey, children }: StepTransitionProps) {
  const shouldReduceMotion = useReducedMotion()

  const variants = {
    initial: { opacity: 0, x: shouldReduceMotion ? 0 : 20 },
    animate: { opacity: 1, x: 0 },
    exit: { opacity: 0, x: shouldReduceMotion ? 0 : -20 },
  }

  return (
    <AnimatePresence mode="wait">
      <motion.div
        key={stepKey}
        variants={variants}
        initial="initial"
        animate="animate"
        exit="exit"
        transition={{
          duration: shouldReduceMotion ? 0 : 0.25,
          ease: [0.25, 0.1, 0.25, 1],
        }}
      >
        {children}
      </motion.div>
    </AnimatePresence>
  )
}
```

Note: Do NOT wrap in overflow:hidden container -- it clips dropdown menus and tooltips within steps. The mode="wait" on AnimatePresence already prevents simultaneous rendering of old+new steps.
